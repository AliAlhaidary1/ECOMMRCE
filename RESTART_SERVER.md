# ⚠️ مهم: إعادة تشغيل السيرفر

## المشكلة الحالية:
السيرفر لا يزال يستخدم إعدادات DATABASE_URL القديمة.

## الحل:

### 1. أوقف السيرفر الحالي:
- اضغط `Ctrl + C` في Terminal الذي يعمل فيه `npm run dev`

### 2. أعد تشغيل السيرفر:
```bash
npm run dev
```

### 3. تحقق من أن الاتصال يعمل:
- افتح الصفحة الرئيسية
- يجب أن تظهر المنتجات الآن
- جرب تسجيل الدخول

---

## إذا استمرت المشكلة:

### أ. تحقق من ملف .env:
```bash
# تأكد من أن DATABASE_URL موجود وصحيح
cat .env
```

### ب. أعد توليد Prisma Client:
```bash
npm run db:generate
```

### ج. اختبر الاتصال:
```bash
npm run db:test-connection
```

### د. إذا كان كل شيء يعمل في الاختبار لكن لا يعمل في السيرفر:
1. احذف مجلد `.next`:
   ```bash
   rm -rf .next
   # أو في Windows:
   rmdir /s .next
   ```

2. أعد تشغيل السيرفر:
   ```bash
   npm run dev
   ```

---

## ملاحظة:
Next.js يحمل متغيرات البيئة عند بدء التشغيل فقط. لذلك يجب إعادة تشغيل السيرفر بعد أي تغيير في ملف `.env`.
